Settings Endpoints

Many of Melon's settings can be changed via an Endpoint, but not all of them. Currently, the following settings are exposed via an Endpoint:

Melon Scanner Settings

  • View Library Paths
  • Set Library Paths
  • View Artist Split indicators
  • Set Artist Split indicators
  • View Genre Split Indicators
  • Set Genre Split Indicators

Plugin Settings

  • View Plugins
  • Disable Plugin
  • Enable Plugin
  • Rescan Plugins folder
  • Reload Plugins

Database Settings

  • Backup Database
  • View Backups
  • Load from backup
  • Reset DB
  • Export Playlist
  • Import Playlist
  • Queue Cleanup Frequency

Updates

  • Check For Updates
  • Update Melon

Missing Settings

The following settings are not exposed via an endpoint, but can be changed via the Console Menu:

Plugins

  • Plugins can expose their own settings UI, but have no standardized way of exposing endpoints for settings. Plugins can make settings endpoints, but unless the client suppose plugins too, default Melon clients wont know about them.
    • In the future, I'd like to make a standardized way for plugins to expose settings.

Network Settings

These settings affect the Melon server, or require a server restart.

  • Edit MongoDb Connection
  • Edit Listening URL
  • Configure HTTPS

Console Customization

  • The console has settings for language and console colors, but these are not exposed with an endpoint.
